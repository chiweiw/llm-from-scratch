需求文档：国际要闻“反重力”AI汇总机器人 (V1.0)1. 项目背景与目标针对当前信息过载、假新闻（如“火云”类截图）频发的现状，构建一个基于 GitHub Actions + Gemini AI 的全自动机器人。核心目标：每天定时抓取路透社、财新等权威信源，通过 AI 浓缩为极简关键词清单。设计原则：只给结论，不给正文；多源校验，剔除谣言。2. 系统功能与架构系统将分为四个模块：数据抓取、信息清洗、AI 处理、终端推送。模块详细功能要求信源接入 (Sources)1. 路透社 (Reuters)：通过 RSS 获取全球第一手快讯。2. 财新一线 (Caixin)：通过 RSSHub 获取中国视角的深度财经/政治动向。自动化逻辑 (Automation)1. 定时执行：每天北京时间 08:00 (Cron: 0 0 * * *) 自动触发。2. 环境隔离：运行在 GitHub Actions 的 Ubuntu 虚拟机上，零本地内存占用。AI 处理 (Summarizer)1. 多源校对：若多个信源提及同一事件，标记为“核心要闻”。2. 关键词提取：利用 Gemini API 将长标题浓缩为 [主语] - [动作/状态] 格式。推送通知 (Delivery)1. 通道：Telegram Bot 或 PushDeer (微信)。2. 格式：Markdown 列表，禁止包含任何摘要段落。3. 非功能性需求 (NFR)安全性 (Security)：API Key 必须存储在 GitHub Repository Secrets 中，代码中严禁硬编码。稳定性 (Reliability)：需具备简单的报错重试机制（如抓取路透社失败时尝试 3 次）。成本控制 (Cost)：全流程 0 成本（利用 GitHub 免费额度 + Gemini 免费层 API）。4. 业务逻辑流程图 (User Story)触发：GitHub Actions 监测到时间到达北京时间早 8:00。抓取：脚本访问 https://www.reuters.com/arc/outboundfeeds/rss/...脚本访问 https://rsshub.app/caixin/mobi/news清洗：提取过去 24 小时内的文章标题，去重。调用 AI：Prompt: *"你是新闻精简专家。将以下标题浓缩为关键词列表。格式：[国家/人物] - [事件]。严禁输出正文。"#发送：将 AI 返回的结果发送至用户的移动端。5. 项目路线图 (Roadmap)Phase 1 (MVP)：实现路透社单源抓取并通过 Telegram 推送关键词。Phase 2 (稳定性)：加入财新作为第二源，实现“双源校验”逻辑（两边都提到了才标记为 Hot）。Phase 3 (反伪造)：增加“假新闻黑名单”过滤逻辑，自动标记可疑信源。
